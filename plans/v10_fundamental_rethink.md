# V10：根本性反思——为什么所有 Gate 方案都失败了？

## 一、已尝试方案的完整清单

| 版本 | Gate 设计 | 门控类型 | 训练 loss | lm_eval |
|------|----------|---------|----------|---------|
| V1 | MLP([h, o, mag, dir]) → α, β | 独立 sigmoid | ≈ 标准 | 发散 |
| V5.x | hidden-size attention → α, β | 独立 sigmoid | ≈ 标准 | 发散 |
| V5.2 | hidden-size attention + RMSNorm → α, β | softmax | ≈ 标准 | 发散 |
| 标准 Qwen3 | 无 gate（h + o） | 无 | 基准 | 基准 |

**所有 gate 方案都有相同的失败模式**：训练 loss 与标准 Qwen3 一致，但 lm_eval 严重发散。

## 二、这个失败模式意味着什么？

### 2.1 训练 loss 正常 = gate 没有破坏局部预测能力

在 teacher-forcing 下，模型给定正确的前 t-1 个 token 预测第 t 个 token。这主要依赖于最近几层的信息。

gate 没有破坏这个能力，说明 gate 在训练过程中学到了"大部分时候接近标准残差"的行为。

### 2.2 lm_eval 发散 = gate 破坏了某种全局一致性

lm_eval 的 lambada 任务需要模型在给定完整上下文后预测最后一个词。perplexity 从 3,498 暴涨到 4,441,009（差 1270 倍），说明模型的**概率分布完全崩溃**了。

这不是"稍微差一点"，而是"完全不工作"。

### 2.3 关键问题：为什么训练时正常但推理时崩溃？

**假设 A：gate 在训练和推理时的行为不同**

如果 gate 的行为依赖于 batch statistics 或 training-specific 的东西（如 dropout），那训练和推理时的行为可能不同。

但 ResidualGate 没有 dropout，也没有 batch normalization。所有操作在训练和推理时应该是一致的。

**假设 B：gate 导致了微小的数值偏移，在 autoregressive 生成中被放大**

在 autoregressive 生成中，每个 token 的预测依赖于之前所有 token 的预测。如果 gate 引入了微小的数值偏移（比如 h_new ≈ 0.99·h + 0.99·o 而不是 h + o），这个偏移会在多步生成中累积。

但 lm_eval 的 lambada 不是 autoregressive 生成——它是给定完整上下文后计算最后一个词的 perplexity。这是一次 forward pass，不涉及多步生成。

**等等——lambada 的 perplexity 是通过一次 forward pass 计算的！**

这意味着问题不是 autoregressive 累积误差，而是**单次 forward pass 就已经崩溃了**。

### 2.4 重新理解：单次 forward pass 就崩溃

lambada perplexity = 4,441,009 意味着模型对正确答案赋予了极低的概率（约 e^{-15.3}）。

这说明在单次 forward pass 中，经过 28 层 × 2 个 gate = 56 次门控后，模型的输出分布已经完全偏离了正确答案。

**这是信息衰减的直接证据**：底层 embedding 的信息在经过 56 次 α < 1 的衰减后丢失了，导致顶层无法正确预测。

### 2.5 但训练 loss 为什么正常？

训练 loss 是在**所有 token 位置**上的平均。大部分 token 的预测只需要局部信息（最近几层的输出），不需要底层 embedding。

lambada 测试的是**特定的、需要长距离依赖的 token**。这些 token 在训练集中占比很小，对平均 loss 的影响微乎其微。

**结论**：gate 破坏了长距离信息传递，但这在平均 loss 中不明显，只在需要长距离依赖的评估任务中暴露。

## 三、根本性问题：α < 1 的递推必然导致信息衰减

### 3.1 数学证明

无论 gate 的设计多么精巧，只要是 `h_{l+1} = α_l · h_l + β_l · o_l` 的形式，且 α_l ∈ (0, 1)：

```
h_L 中 h_0 的贡献 = (∏_{l=1}^{L} α_l) · h_0
```

对于 sigmoid 输出：α_l ∈ (0, 1)，所以 ∏ α_l → 0 当 L → ∞。

即使 α_l 的期望值很接近 1（比如 0.99），经过 56 次递推：0.99^56 ≈ 0.57。

这意味着 h_0 的信息被衰减了 43%。对于 lambada 这种需要精确语义理解的任务，43% 的信息损失可能就是致命的。

### 3.2 为什么标准残差不会有这个问题？

标准残差 `h_{l+1} = h_l + o_l`，α 恒为 1：

```
h_L 中 h_0 的贡献 = 1^L · h_0 = h_0
```

h_0 的信息完整保留。

### 3.3 AttnRes 为什么不会有这个问题？

AttnRes 不是递推形式，而是**直接聚合**：

```
h_L = Σ_{i=0}^{L-1} α_{i→L} · v_i
```

每一层直接从所有历史层获取信息，不经过中间层的递推。h_0 的信息可以直接传到 h_L，不会被中间层衰减。

## 四、根本性的设计约束

从以上分析，我们得到一个**不可回避的约束**：

> **在递推形式 `h_{l+1} = f(h_l, o_l)` 中，如果 f 对 h_l 的系数 < 1，信息必然衰减。**

这意味着：

1. **不能用 sigmoid/softmax 来缩放 h**：因为它们的输出 < 1
2. **不能用任何 (0, 1) 范围的门控来缩放 h**：同理
3. **如果要过滤 h 中的冗余信息，不能通过缩放来实现**

## 五、那么如何在不缩放 h 的情况下过滤冗余信息？

### 5.1 方向一：投影而非缩放

不缩放 h 的幅度，而是**旋转 h 的方向**。

```
h_new = R(h, o) · h + o
```

其中 R 是一个接近单位矩阵的旋转/投影矩阵。

- R = I 时：标准残差
- R ≠ I 时：h 的方向被调整，冗余维度的信息被旋转到有用维度

**关键**：R 保持 h 的幅度不变（正交矩阵），所以不会导致信息衰减。

但这需要 d×d 的矩阵，参数量太大。低秩近似：R = I + U·V^T。

### 5.2 方向二：加法修正而非乘法缩放

不缩放 h，而是**加一个修正项**来抵消冗余信息。

```
h_new = h + o + correction(h, o)
```

其中 correction 是一个小的修正向量，用于抵消 h 中的冗余部分。

```
correction = -gate · project_redundant(h)
```

- gate ∈ [0, 1]：控制修正强度
- project_redundant(h)：提取 h 中的冗余分量

**关键**：h 的主分量始终保留（系数为 1），只有冗余分量被减去。

### 5.3 方向三：非递推形式——Block AttnRes

AttnRes 论文本身提出了 Block AttnRes（Figure 1c），将层分成 block，在 block 内使用标准残差，在 block 之间使用 attention 聚合。

这是一个**已被验证有效**的方案。

```
Block 1: layers 1-7, 标准残差
Block 2: layers 8-14, 标准残差
Block 3: layers 15-21, 标准残差
Block 4: layers 22-28, 标准残差

Block 之间：attention 聚合
h_block_2 = Σ α_{i→2} · v_block_i    (在 block 级别做 attention)
```

### 5.4 方向四：将 hidden-size attention 用于修正而非替代

**核心思想**：保留标准残差连接，但在其基础上加一个 hidden-size attention 驱动的修正项。

```
h_new = h + o + δ(h, o)
```

其中 δ 是 hidden-size attention 计算的修正向量：

```
# 用 attention 机制分析 h 和 o 的关系
Q = compress(h)
K = compress(o)
score = Q · K / √r

# 修正向量：基于 attention score 调制 o
δ = score · W_correction · o

# 或者：修正向量用于调整 h 中的冗余部分
δ = -score · W_correction · h
```

**关键**：h 和 o 都以系数 1 保留（标准残差），δ 只是一个小的修正。

这保证了：
- 梯度直通：∂h_new/∂h = I + ∂δ/∂h ≈ I
- 信息不衰减：h 的系数恒为 1
- 冗余过滤：δ 可以抵消 h 中的冗余部分

### 5.5 方向五：重新参数化——让 α 的期望值恒为 1

如果坚持 `h_new = α · h + β · o` 的形式，可以重新参数化 α 使其期望值为 1：

```
α = 1 + tanh(gate_logit) · scale    # 范围 (1-scale, 1+scale)
```

当 scale = 0.1 时，α ∈ (0.9, 1.1)。

经过 56 次递推：
- 最坏情况：0.9^56 ≈ 0.003（仍然衰减）
- 期望情况：1.0^56 = 1.0（不衰减）
- 最好情况：1.1^56 ≈ 227（爆炸）

**问题**：α > 1 会导致幅度爆炸。

**解决**：配合 RMSNorm。每层前的 RMSNorm 会将幅度归一化，所以即使 α 略 > 1，幅度也不会无限增长。

```
h_{l+1} = α_l · h_l + β_l · o_l
h_{l+1}_norm = RMSNorm(h_{l+1})    ← 下一层的输入
```

RMSNorm 会将 h_{l+1} 的幅度归一化到固定尺度，所以 α > 1 不会导致数值爆炸。

**但 RMSNorm 也会消除 α < 1 的衰减效果**——因为 RMSNorm 会将缩小的向量重新放大。

**等等，这意味着 RMSNorm 应该能补偿 α < 1 的衰减！**

### 5.6 重新分析：RMSNorm 是否补偿了 α < 1 的衰减？

在 Pre-Norm Transformer 中：

```
h_{l+1} = α · h_l + β · o_l
input_to_sublayer = RMSNorm(h_{l+1})
o_{l+1} = SubLayer(input_to_sublayer)
```

RMSNorm 将 h_{l+1} 归一化到固定幅度。所以即使 α < 1 导致 h_{l+1} 的幅度缩小，RMSNorm 会将其放大回来。

**但 RMSNorm 只恢复了幅度，没有恢复方向信息！**

如果 α · h 和 β · o 的混合改变了 h 的方向（将有用信息的方向旋转到了噪声方向），RMSNorm 无法恢复这个方向信息。

**所以问题不是幅度衰减，而是方向信息的丢失。**

### 5.7 方向信息丢失的机制

```
h_new = α · h + β · o
```

当 α < 1 时，h 的方向分量被缩小，o 的方向分量相对增大。这改变了 h_new 的方向。

经过多层后，h_new 的方向越来越偏向最近几层的 o，而远离底层的 h_0。

**这就是为什么 lambada 失败**：底层 embedding 的方向信息在深层被 o 的方向信息覆盖了。

## 六、最终洞察

### 6.1 问题的本质

**不是幅度衰减（RMSNorm 可以补偿），而是方向信息的覆盖。**

当 α < 1 时，h 的方向贡献被削弱，o 的方向贡献被增强。经过多层后，残差流的方向越来越由最近几层的 o 主导，底层的方向信息被覆盖。

### 6.2 标准残差为什么不会有这个问题？

`h_new = h + o`：h 和 o 的方向贡献是等权的。h 的方向信息不会被 o 覆盖——它们是叠加的。

虽然 h 的幅度会随层数增长（因为不断叠加 o），但 RMSNorm 会归一化幅度。关键是 h 的方向信息始终保留在残差流中。

### 6.3 如何在过滤冗余的同时保留方向信息？

**方案：在子空间级别做选择，而不是在幅度级别做缩放**

将 hidden-size 分成多个子空间。对于每个子空间，做一个二选一的决策：保留 h 的子空间还是用 o 的子空间替换。

```
h_new = concat([
    select(h_1, o_1),    # 子空间 1：选择 h 或 o
    select(h_2, o_2),    # 子空间 2：选择 h 或 o
    ...
    select(h_G, o_G),    # 子空间 G：选择 h 或 o
])
```

其中 select 是一个硬选择（或软选择）：

```
select(h_g, o_g) = gate_g · o_g + (1 - gate_g) · h_g
```

**关键区别**：这里的 gate 是在**子空间级别**做选择，而不是在**幅度级别**做缩放。

当 gate_g = 0 时：保留 h_g（h 的方向信息完整保留）
当 gate_g = 1 时：用 o_g 替换 h_g（h 的方向信息被替换，不是被衰减）

**这仍然是 α < 1 的形式**（α = 1 - gate），所以仍然会有方向信息丢失。

### 6.4 真正的解决方案

也许问题不在于 gate 的设计，而在于**在每一层都做门控**这个策略本身。

**AttnRes 的成功不是因为它在每一层做了更好的门控，而是因为它改变了信息聚合的拓扑结构**——从串行递推变成了并行聚合。

在串行递推中，信息必须经过所有中间层才能从底层传到顶层。任何中间层的衰减都会累积。

在并行聚合中，信息可以直接从底层传到顶层，不经过中间层。

**结论**：如果要在 hidden-size 维度上实现 AttnRes 的效果，不能用递推形式（每层独立门控），而必须用某种形式的并行聚合或跳跃连接。

### 6.5 可能的方向

1. **Block AttnRes**（论文已验证）：在 block 之间做 attention 聚合
2. **稀疏跳跃连接**：让深层可以直接访问浅层的输出（不经过中间层）
3. **辅助残差流**：维护一个不经过门控的辅助残差流，与门控残差流并行
4. **门控只用于 o，不用于 h**：`h_new = h + gate · o`，但 gate 的范围包含负值（可以减去 o 的有害部分）

**方向 4 的变体**：

```
h_new = h + gate · o + correction
```

其中 correction 是一个小的修正向量，用于抵消 h 中的冗余部分：

```
correction = -small_gate · project(h)
```

project(h) 提取 h 中与 o 冗余的部分，small_gate 控制修正强度。

这保证了 h 的主分量始终保留（系数为 1），只有冗余分量被减去。
